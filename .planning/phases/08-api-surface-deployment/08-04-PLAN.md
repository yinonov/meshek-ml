## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| HTTP client → pydantic (RecommendRequest) | Untrusted `merchant_id` crosses here; regex-validated at 422 |
| `app.state.engine` | Built once at startup; never mutated; safe for concurrent sync-threadpool reads |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 (inherited) | Tampering | merchant_id path traversal via recommend | mitigate | `MerchantIdStr` regex on `RecommendRequest.merchant_id`. |
| T-8-08 | Information Disclosure | Exception from `engine.recommend()` leaking stack traces or model internals | mitigate | Exceptions bubble to plan 05's central handler which returns an opaque `request_id` and never exposes the traceback in the response body. |
| T-8-09 | Availability | Degraded-start Tier 3 raises RuntimeError on every /recommend call for a merchant with ≥14 days of history | accept | This is the intended behavior — Fly.io health check returns 503 so the orchestrator can reroute. Documented in integration notes. |
</threat_model>
