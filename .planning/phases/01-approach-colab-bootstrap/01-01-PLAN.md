**Section: `## Method Decisions`** with the following subsections, each containing a "Why this method", "Why not alternatives", and "Supporting evidence" structure:

### 1. Demand Forecasting: LightGBM
- Why: M5 competition top-5 all used LightGBM; handles mixed features, missing values, non-linear relationships naturally
- Evidence: P4 (Lee & Wong) confirms tree models outperform LSTM on MAE/RMSE/MAPE/R-squared for perishable categories; P7 (MARIOD) uses TFT but requires 8x A100 GPUs — infeasible for Colab
- Not chosen: ARIMA (P2, P8 use it but limited to linear relationships), LSTM (P6 uses attention-enhanced LSTM but needs large datasets), TFT (P7 — too resource-heavy)

### 2. Inventory Optimization: PPO
- Why: Handles sequential decision-making with continuous action space; stable training; Stable-Baselines3 ecosystem
- Evidence: Boute et al. 2022 roadmap paper; P4 (Lee & Wong) compares DRL favorably against time series and random forest for inventory; P3 (Zheng et al.) validates DRL for replenishment decisions
- Not chosen: DQN (discrete actions only — coarse ordering), SAC (higher complexity, marginal gain per P4)

### 3. Analytical Baseline: Newsvendor
- Why: Classical single-period model directly maps to daily perishable ordering; critical fractile formula is interpretable; natural bridge between forecast and order decision
- Evidence: All 8 papers reference newsvendor; P5 (Liao et al.) embeds it as structural prior in E2E-PIL; P8 uses it as baseline for ARIMA-driven restocking

### 4. Architecture: Two-Stage (Forecast then Optimize)
- Why: Simpler to debug, matches existing codebase separation, each component testable independently
- Evidence: P5 (Liao et al.) shows E2E-PIL outperforms two-stage but acknowledges two-stage as standard starting point; P4 (Lee & Wong) uses two-stage approach successfully
- Future: E2E with embedded newsvendor structure (P5's E2E-PIL) is the natural v2 upgrade

### 5. Simulation as Training Ground
- Why: Real data scarce for small merchants; synthetic data enables controlled experiments; simulation validates pipeline before real data
- Evidence: P5 (Liao et al.) uses both synthetic and real data experiments; P7 (MARIOD) validates on simulated sensor data; original research brief confirms this approach

**Section: `## Explicitly Deferred`** (addresses APPR-02) with subsections:

### Dynamic Pricing
- What: Price as a coupled action variable alongside ordering quantity
- Why deferred: P3 (Zheng et al.) proves joint profit is not jointly concave — requires specialized dual-agent architecture; P1 (Xia et al.) and P6 (Liu et al.) both couple pricing with inventory but add significant complexity
- When: v2, after single-merchant pipeline is stable; price field recorded in data schema for forward compatibility

### End-to-End Forecast-Optimize
- What: Train neural network directly with inventory cost function instead of separate forecast and optimize stages
- Why deferred: P5 (Liao et al.) shows E2E-PIL outperforms two-stage, but two-stage is simpler to debug and matches current code separation
- When: v2, building on stable two-stage baseline

### Federated Learning
- What: Flower-based federated training across heterogeneous merchants
- Why deferred: Novel contribution of the project but needs stable single-merchant pipeline first; separate milestone
- When: After this milestone completes successfully

### Attention-Based Interpretability
- What: Attention mechanisms for demand driver analysis
- Why deferred: P1 (TCN-Attention), P6 (LSTM-Attention), P7 (Transformer attention) all use attention but require deep learning models; LightGBM uses feature importance instead
- When: If/when deep learning forecasting models are added
