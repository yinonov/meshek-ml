<objective>
Make MerchantStore safe for GCS FUSE persistence by forcing SQLite out of WAL mode
before any migrations run, and stand up the tests/deploy package scaffolding that
subsequent plans (02, 04) will populate.

Purpose: GCS FUSE does not honor POSIX file locking. SQLite's default WAL mode uses
mmap'd sidecar files (`.wal`, `.shm`) with atomic rename semantics that FUSE silently
mishandles. Forcing `journal_mode=DELETE` + `synchronous=FULL` eliminates WAL entirely
and falls back to the classic rollback journal, which FUSE handles reliably. This is
D-10/D-11 and the single most important gotcha in Phase 8.1 (RESEARCH §PRAGMA).

Output: Regression test file asserting the two PRAGMA states, the storage module edit,
and an empty-but-collectable tests/deploy package.
</objective>
