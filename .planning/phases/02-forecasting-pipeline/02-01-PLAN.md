```python
"""Forecasting train/eval orchestrator."""

from __future__ import annotations

from typing import TYPE_CHECKING, Any

import pandas as pd

from meshek_ml.common.seed import set_global_seed
from meshek_ml.forecasting.evaluation import compute_all_metrics
from meshek_ml.forecasting.features import (
    add_calendar_features,
    add_lag_features,
    add_rolling_features,
)
from meshek_ml.forecasting.schema import (
    SchemaValidationError,
    normalize_simulation_data,
    validate_demand_schema,
)
from meshek_ml.forecasting.tree_models import train_lightgbm, train_xgboost

if TYPE_CHECKING:
    from meshek_ml.common.types import MetricsDict

TARGET_COL = "quantity"

FEATURE_COLS_TO_DROP = ["date", "merchant_id", "product", "quantity"]

MODEL_REGISTRY: dict[str, Any] = {
    "lightgbm": train_lightgbm,
    "xgboost": train_xgboost,
}
```

Implement `run_forecast_pipeline(data, model_type="lightgbm", train_end_date="2024-06-30", horizon=7, seed=42) -> MetricsDict`:

Step-by-step logic:
1. `set_global_seed(seed)` for reproducibility.
2. Copy data. If `"realized_demand"` in columns (simulation data), call `normalize_simulation_data(data)` to map to canonical schema. This implements FORE-04 -- same code path after normalization.
3. Call `validate_demand_schema(df)` -- this implements FORE-05 fail-fast.
4. Ensure `df["date"]` is datetime via `pd.to_datetime`.
5. Sort by `["merchant_id", "product", "date"]`.
6. Apply feature engineering (passing `target_col=TARGET_COL` to lag and rolling functions):
   - `df = add_lag_features(df, target_col=TARGET_COL)`
   - `df = add_rolling_features(df, target_col=TARGET_COL)`
   - `df = add_calendar_features(df)`
7. Time-based split (FORE-02): `train_mask = df["date"] <= pd.Timestamp(train_end_date)`. Train = `df[train_mask]`, val = `df[~train_mask]`.
8. Drop rows with NaN (from lag features) in train and val separately: `train = train.dropna()`, `val = val.dropna()`.
9. If val is empty after dropping NaN, raise `ValueError("No validation data after removing NaN rows. Check train_end_date or data range.")`.
10. Build feature matrix: `feature_cols = [c for c in train.columns if c not in FEATURE_COLS_TO_DROP]`. `x_train = train[feature_cols]`, `y_train = train[TARGET_COL].values`, `x_val = val[feature_cols]`, `y_val = val[TARGET_COL].values`.
11. Look up model trainer: `train_fn = MODEL_REGISTRY[model_type]`. If not found, raise `ValueError(f"Unknown model_type: {model_type}. Choose from {list(MODEL_REGISTRY)}")`.
12. `model = train_fn(x_train, y_train)`
13. `predictions = model.predict(x_val)`
14. `metrics = compute_all_metrics(y_val, predictions)` -- this returns MAE, RMSE, WMAPE, pinball_loss (FORE-03).
15. Return metrics.

Also add a convenience function `load_and_run(source, path=None, model_type="lightgbm", train_end_date="2024-06-30", seed=42) -> tuple[pd.DataFrame, MetricsDict]`:
- If `source == "synthetic"`: call `run_simulation(n_merchants=3, start_date="2024-01-01", end_date="2024-12-31", seed=seed)` to generate data.
- If `source == "csv"`: call `load_csv(path)`.
- If `source == "parquet"`: call `load_parquet(path)`.
- Else raise `ValueError(f"Unknown source: {source}")`.
- Call `run_forecast_pipeline(data, model_type, train_end_date, seed=seed)`.
- Return `(data, metrics)`.
