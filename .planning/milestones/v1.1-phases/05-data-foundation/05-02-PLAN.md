6. **Migrations (Pattern 2):**
   ```python
   def _migration_001_initial(conn: sqlite3.Connection) -> None:
       conn.executescript("""
           CREATE TABLE IF NOT EXISTS merchant_profile (
               merchant_id TEXT PRIMARY KEY NOT NULL,
               name        TEXT,
               timezone    TEXT NOT NULL DEFAULT 'Asia/Jerusalem',
               language    TEXT NOT NULL DEFAULT 'he',
               created_at  TEXT NOT NULL
           );
           CREATE TABLE IF NOT EXISTS sales (
               date        TEXT NOT NULL,
               merchant_id TEXT NOT NULL,
               product     TEXT NOT NULL,
               quantity    REAL NOT NULL,
               PRIMARY KEY (date, product)
           );
           CREATE INDEX IF NOT EXISTS idx_sales_product ON sales(product);
       """)

   _MIGRATIONS = [(1, _migration_001_initial)]

   def _apply_migrations(conn: sqlite3.Connection) -> None:
       current = conn.execute("PRAGMA user_version").fetchone()[0]
       for target, fn in _MIGRATIONS:
           if current < target:
               with conn:
                   fn(conn)
                   conn.execute(f"PRAGMA user_version = {target}")
               current = target
   ```
   Add a NOTE comment: "PRAGMA user_version is the application slot. PRAGMA schema_version is SQLite's internal cache invalidation counter — never touch it (Pitfall 3)."

7. **`MerchantStore` class:**

     - validate id → store on `self.merchant_id`

     - if `must_exist and not self._path.exists()` → raise `UnknownMerchantError(f"No store for merchant {merchant_id!r}")`
     - `self._path.parent.mkdir(parents=True, exist_ok=True)`
     - `self._conn = sqlite3.connect(str(self._path), detect_types=sqlite3.PARSE_DECLTYPES)`
     - `self._conn.execute("PRAGMA foreign_keys = ON")`
     - `self._conn.execute("PRAGMA journal_mode = WAL")`
     - `_apply_migrations(self._conn)`
   - `__enter__` returns self; `__exit__` calls `self.close()`.
   - `close()`: closes the connection if not already closed; idempotent; sets `self._conn = None`.
   - `create_profile(profile: MerchantProfile) -> MerchantProfile`: enforce `profile.merchant_id == self.merchant_id` (else `InvalidMerchantIdError`); `with self._conn:` parameterized INSERT into `merchant_profile`; return profile.

     - `with self._conn: self._conn.executemany("INSERT INTO sales (date, merchant_id, product, quantity) VALUES (?, ?, ?, ?) ON CONFLICT(date, product) DO UPDATE SET quantity = excluded.quantity, merchant_id = excluded.merchant_id", rows)`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| caller (Phase 8 FastAPI / future meshek app) → meshek-ml Python API | `merchant_id` is caller-supplied (D-14) and becomes both a filename (D-01) and a SQL parameter. This is the entire attack surface. |
| meshek-ml Python API → SQLite file | parameterized SQL only; no string interpolation of caller data into queries. |
| meshek-ml Python API → filesystem | Path construction validated by regex whitelist + `Path.resolve()` parent check. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-5-01 | Tampering / Elevation of Privilege | `MerchantStore.__init__` → `_merchant_path` (filename construction from caller-supplied `merchant_id`) | mitigate | (1) `_MERCHANT_ID_PATTERN = re.compile(r"^[A-Za-z0-9_-]{1,64}$")` rejects `/`, `\`, `.`, `\x00`, unicode, oversized, empty. (2) After `Path.resolve()`, verify `candidate.parent == _data_root()` — defense in depth in case the regex is ever loosened. Both defenses asserted by parametrized tests in `tests/storage/test_path_traversal.py` (Plan 01). |
| T-5-02 | Tampering (SQL Injection) | All SQL in `MerchantStore` (profile + sales DML, range queries) | mitigate | Every value passes through `?` placeholders. The only f-string SQL allowed is `PRAGMA user_version = {target_version}` where `target_version` is a hardcoded `int` constant in `_MIGRATIONS`. Acceptance criterion enforces `grep -nE 'f"(INSERT\|SELECT\|UPDATE\|DELETE)'` returns nothing. |
| T-5-03 | Information Disclosure (cross-merchant data bleed) | `write_sales` accepting a DataFrame with foreign `merchant_id` rows | mitigate | `write_sales` rejects any row whose `merchant_id` differs from `self.merchant_id` with `SchemaValidationError`. Tested by `test_write_rejects_foreign_merchant_id` in Plan 01. Filesystem isolation (D-01) provides the second layer. |
| T-5-04 | Denial of Service (disk fill via huge DataFrame) | `write_sales` executemany | accept | v1.1 volumes are tens of rows/day per merchant. Caller is the trusted meshek app, not direct internet input. Revisit if Phase 8 exposes /sales without rate limiting. |
</threat_model>
