<integration_notes>
**Degraded-startup contract (research finding #4 — planner-owned design choice):**
The Phase 6 `build_lifespan()` intentionally raises `RuntimeError` if the model file is missing (it was written to fail fast for CLI usage). Phase 8's `create_app()` wraps this lifespan in an outer `asynccontextmanager` that catches that `RuntimeError` at enter and proceeds with `app.state.ml = None`. This gives us three wins:
1. `GET /health` can deterministically return 503 without contrived test setup.
2. Tier 1 / Tier 2 recommendations (which do not need the model) still work when the model bundle is absent or corrupt.
3. Fly.io / Railway health checks observe 503 immediately, so a bad model deploy is externally visible without a full crash loop.

Plan 04 (recommend router) will extend this wrapper to also populate `app.state.engine` and `app.state.catalog`, and will respect the same degraded contract (engine built with `model=None`, `residual_std=0.0`, `feature_cols=[]` when `ml is None`).

**Shared schema side-effect:** `schemas.py` is introduced in this plan because `HealthResponse` needs it, and plans 02/03/04 will append `MerchantIdStr`, `CreateMerchantRequest`, `SalesRequest`, etc. to the same file. `MerchantIdStr` (the regex-validated type) is defined here in Wave 1 because plan 02 depends on it transitively via the same file; shipping it now avoids plan 02 touching this file for a stub import.
</integration_notes>
