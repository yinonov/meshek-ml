       - `MESHEK_TRAIN_N_MERCHANTS` (default `20`)

Bundle load verification: inline `python -c` block that sets `MESHEK_MODELS_DIR` and calls `load_model_bundle(Path("${BUNDLE_PATH}"))`, asserting `feature_cols` non-empty and `residual_std > 0`. Print a one-line summary on success. **Reuses the existing path-guarded loader — do not reimplement verification in bash.**
    7. `LOCAL_ONLY=1` branch: echo "LOCAL_ONLY=1 — skipping GCS upload" and `exit 0`.
    8. `DRY_RUN=1` branch: `printf '+ gcloud storage cp --cache-control=no-cache %s %s\n' "${BUNDLE_PATH}" "${GCS_DEST}"` then `exit 0`. The `gcloud storage cp` literal MUST appear in stdout so task 9-02-01's automated grep passes.
    9. Full mode:
       - `gcloud storage cp --cache-control=no-cache "${BUNDLE_PATH}" "${GCS_DEST}"` (D-12 — `--cache-control=no-cache` is mandatory to avoid FUSE cold-start reading stale bytes)
       - Read generation number: `gcloud storage objects describe "${GCS_DEST}" --format='value(generation)'` (research §"Code Examples" uses `objects list`; `describe` is equivalent and semantically clearer — either is acceptable; see research A1 for the verified format spec).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator shell → filesystem | Script creates `BUNDLE_DIR` and writes the bundle. Owned by the operator; traversal guard via `MESHEK_MODELS_DIR` export. |
| operator shell → GCS | Full-mode `gcloud storage cp` runs with the operator's ambient gcloud credentials. The service account has only `objectViewer` (D-05) — it cannot upload. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-9-02-01 | Tampering | `BUNDLE_PATH` env var accepted from caller | mitigate | `export MESHEK_MODELS_DIR="$(cd "$(dirname "$BUNDLE_PATH")" && pwd)"` anchors the traversal guard to the resolved dirname; Phase 6 T-6-08 guard in `save_model_bundle`/`load_model_bundle` rejects anything outside this root. |
| T-9-02-02 | Spoofing | Uploaded bundle bypasses review | accept | Only operators with GCP write credentials on `gs://meshek-prod-models` can upload; enforced at the cloud IAM layer, not inside the script. Documented in D-23. |
| T-9-02-03 | Denial of Service | Misconfigured `DAYS` / `N_MERCHANTS` | accept | Operator-only script, not exposed to network. Large params only burn local CPU. |
</threat_model>
