<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator shell → filesystem | cli_train writes the bundle under `MESHEK_MODELS_DIR`; the traversal guard in `train_and_save`/`model_io` is the only defense against an output path escaping the root. |
| pytest → filesystem | tests write bundles to `tmp_path`; monkeypatched `MESHEK_MODELS_DIR` must point at `tmp_path` or the traversal guard will (correctly) reject the write. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-9-01-01 | Tampering | cli_train.py `--output` flag | mitigate | Existing `_assert_within_root()` in `model_io.save_model_bundle` rejects any path outside `MESHEK_MODELS_DIR`. Phase 6 T-6-08 traversal guard still applies — do NOT weaken it. cli_train.py must NOT set `MESHEK_MODELS_DIR` itself; caller (bash wrapper or test fixture) owns the root. |
| T-9-01-02 | Tampering | deserialisation of bundle in tests | accept | Tests only load bundles they just wrote in the same test run; research §"Pitfall 7" notes deserialisation is acceptable for v1.1 because only operator GCP credentials can write the production bucket. |
| T-9-01-03 | Information Disclosure | JSON stdout summary | accept | Summary contains only synthetic data metadata (row count, residual_std, feature count). No PII, no secrets. |
</threat_model>
