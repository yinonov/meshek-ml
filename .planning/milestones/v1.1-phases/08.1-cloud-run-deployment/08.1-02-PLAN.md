<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Cloud Run runtime → container process | `PORT` env var is injected by the platform and consumed by shell-form CMD |
| Container stdout → Cloud Logging | JSON log lines cross into Google's logging ingestion pipeline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-8.1-02-01 | Information Disclosure | JSONFormatter payload in Cloud Logging | accept | Existing Phase 8 T-8-10 mitigation (opaque request_id, no stack trace in response) already applies to log contents; adding `severity` surfaces only the level name, no new PII |
| T-8.1-02-02 | Denial of Service | Shell-form CMD parsing | accept | Shell-form is a documented, stable Docker feature; the `:-8000` default eliminates the only plausible misconfig (unset PORT crashes container) |
| T-8.1-02-03 | Elevation of Privilege | sh -c invocation inside container | accept | Container still runs as `USER appuser`; shell-form does not escalate, it only forks `/bin/sh -c` which inherits the unprivileged user |
</threat_model>
