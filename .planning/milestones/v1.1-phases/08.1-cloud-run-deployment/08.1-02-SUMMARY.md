# Phase 8.1 Plan 02: Dockerfile $PORT + Cloud Logging Severity Summary

**One-liner:** Shell-form `CMD uvicorn ... --port ${PORT:-8000}` lets Cloud Run inject PORT=8080 while preserving Fly.io's 8000 default, and `severity=record.levelname` in `JSONFormatter` unlocks auto-structured severity coloring in the Cloud Logging UI.

## What Was Built

- **`Dockerfile` line 48–51**: Replaced exec-form `CMD ["uvicorn", ...]` with shell-form `CMD uvicorn meshek_ml.service.app:create_app --factory --host 0.0.0.0 --port ${PORT:-8000}`. EXPOSE 8000, HEALTHCHECK, USER appuser, and all ENV blocks are untouched. Cloud Run's injected `PORT=8080` env var is now consumed; Fly.io gets the `:-8000` default.

- **`src/meshek_ml/service/errors.py` JSONFormatter.format**: Added `"severity": record.levelname` as a second top-level field alongside the existing `"level"` field. Cloud Logging auto-detects the `severity` field and populates the severity column in the logging UI with color-coded levels (DEBUG/INFO/WARNING/ERROR/CRITICAL). Purely additive — no existing field changed.

- **`tests/service/test_errors.py`**: Two new tests added via TDD RED→GREEN:
  - `test_jsonformatter_emits_cloud_logging_severity`: asserts `severity == "ERROR"` and `level == "ERROR"` (legacy preserved) for an ERROR record.
  - `test_jsonformatter_severity_info_case`: asserts `severity == "INFO"` for an INFO record.

## Decisions Made

1. **Shell-form CMD over ENTRYPOINT script**: Shell-form keeps the change to a single line edit with no new file. The `${PORT:-8000}` default explicitly handles both Cloud Run (PORT=8080 injected) and Fly.io (no PORT, falls back to 8000). D-25 locked this as the preferred approach.

2. **`severity` is additive, not a replacement**: Keeping `level` unchanged means any existing log consumers or dashboards that parse the `level` field continue working. The `severity` field is a pure Cloud Logging compatibility addition.
